# Glossary

**Terms, Expressions and Acronyms (TEA) must be organized alphabetically.**


| **_TEA_** (EN)                                    | **_Description_** (EN)                                                                                                                                                    |                                       
|:--------------------------------------------------|:--------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **Admin**                                         | User that manages other users.                                                                                                                                            |
| **Air Space**                                     | Space in the air that the drones can use in order to do the show.                                                                                                         |
| **Central Orchestrator**                          | Central that coordinates the drones to initiate a sequence or receives feedback from the drones regarding the completion of a sequence.                                   |
| **Client Status**                                 | Clients can have status that indicates if they were Deleted, Infringement, Created, Regular or VIP.                                                                       |
| **CRM Collaborator**                              | Acronym for Customer Relationship Management Collaborator.                                                                                                                |
| **CRM Manager**                                   | Acronym for Customer Relationship Management Manager.                                                                                                                     |
| **Customer**                                      | An organization (public or public) requesting a drone show; identified by VAT number and official contact details.                                                        |
| **Customer Representative**                       | A person linked to a customer entity who interacts with Shodrone and may receive proposals.                                                                               |
| **Customer Relationship Management Collaborator** | User that interacts directly with the customer and handles their requests.                                                                                                |
| **Customer Relationship Management Manager**      | User responsible for overseeing customer-related operations and managing client status and proposals.                                                                     |
| **Domain Specific Language**                      | A computer language specialized to a particular application domain.                                                                                                       |
| **Drone**                                         | An unmanned aerial vehicle (UAV) used to form figures and execute shows.                                                                                                  |
| **Drone Collision**                               | The movements of each drone have to be calculated in order to avoid collision between drones.                                                                             |
| **Drone Inventory**                               | The current list of drones available for use, each identified by a serial number and model.                                                                               |
| **Drone Language Plugin**                         | A software plugin used to validate or analyze code written for a specific drone language.                                                                                 |
| **Drone Model**                                   | The type or category of drone, each with its own specifications and programming language.                                                                                 |
| **Drone Program**                                 | A code routine executed by an individual drone to perform movements in a show or figure.                                                                                  |
| **Drone Programming Language**                    | The specific language used to program individual drones based on their model.                                                                                             |
| **Drone Simulator**                               | A system component used to simulate the execution of drone shows and figures to detect errors or collisions.                                                              |
| **Drone Tech**                                    | Technical user who is responsible to handle everything related to drone models, inventory, programming and simulation.                                                    |
| **DSL Plugin**                                    | A plugin used to analyze or validate the DSL figure description code.                                                                                                     |
| **Exclusive Figure**                              | Some figures are exclusive, it means that they are available only to one client.                                                                                          |
| **Figure**                                        | A visual representation formed by coordinated drones in the sky, which may be static or dynamic, and is composed of drone routines written in a domain-specific language. |
| **Figure DSL Code**                               | Code written in a domain-specific language that describes the structure and movement of figures.                                                                          |
| **Library**                                       | A centralized collection of all figures available in the system.                                                                                                          |
| **Plugin**                                        | A software component that adds specific capabilities, like validating templates or drone code.                                                                            |
| **Proposal**                                      | A formal document sent to a client describing the planned drone show, including figures and drones.                                                                       |
| **Proposal Template**                             | A predefined structure used to format show proposals for client presentation.                                                                                             |
| **Sequence**                                      | An ordered set of figures forming part of a show.                                                                                                                         |
| **Shodrone**                                      | Name of the company that this project is solving.                                                                                                                         |
| **Show**                                          | A combination of one or more sequences of figures performed by drones, usually based on a client's request.                                                               |
| **Show Designer**                                 | User that designs the drone show.                                                                                                                                         |
| **Show Proposal**                                 | A document that includes show specifications, figure selection, drone models, and a simulation preview video, sent to the customer for approval.                          |
| **Show Request**                                  | A formal request submitted by a client (new or existing) to schedule a drone show.                                                                                        |
| **Simulation**                                    | An imitative representation of a show so that it can be used to make sure that the show has no problems.                                                                  |
| **Simulation Report**                             | A document summarizing the outcome of a simulated show or figure, including collisions and status.                                                                        |
| **Subareas**                                      | Designated airspace segments assigned to different drones or simulation servers.                                                                                          |
| **Time Step**                                     | A simulation unit of time used to synchronize drone movements.                                                                                                            |
| **3D Matrix**                                     | A spatial representation used to track drone positions and detect collisions during simulation.                                                                           |
