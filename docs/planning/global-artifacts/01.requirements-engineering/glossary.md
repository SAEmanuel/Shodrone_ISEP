# Glossary

**Terms, Expressions and Acronyms (TEA) must be organized alphabetically.**


| **_TEA_** (EN)               | **_Description_** (EN)                                                                                                                  |                                       
|:-----------------------------|:----------------------------------------------------------------------------------------------------------------------------------------|
| **Air Space**                | Space in the air that the drones can use in order to do the show.                                                                       |
| **Central Orchestrator**     | Central that coordinates the drones to initiate a sequence or receives feedback from the drones regarding the completion of a sequence. |
| **Client Status**            | Clients can have status that indicates if they were Deleted, Infringement, Created, Regular or VIP.                                     |
| **Drone Collision**          | The movements of each drone have to be calculated in order to avoid collision between drones.                                           |
| **Domain Specific Language** | A computer language specialized to a particular application domain.                                                                     |
| **Exclusive Figure**         | Some figures are exclusive, it means that they are available only to one client.                                                        |
| **Figure**                   | The drones show make a lot of images in the sky, they are called figures.                                                               |
| **Library**                  | All the figures are stored in a library.                                                                                                |
| **Sequence**                 | A set number of figures.                                                                                                                |
| **Shodrone**                 | Name of the company that this project is solving.                                                                                       |
| **Show Request**             | A request for a show made by someone who can already be a existing client or a new client.                                              |
| **Show**                     | A set of sequences that the drones have to do in order to fulfill a request from a client.                                              |
| **Simulation**               | An imitative representation of a show so that it can be used to make sure that the show has no problems.                                |
| **Subareas**                 | Each drone has their designated areas, this areas are called subareas.                                                                  |
